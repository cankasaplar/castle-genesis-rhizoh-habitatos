## 1. Katman I — Frozen Civilization Kernel

**İçerik (örnek aile):** PCEK, PAG, πEFC, RBL, ETK, HOGA, TLOA, … — **görünmez**, **yavaş değişen**, **denetlenebilir**, **append-only** ve güvenlik / süreklilik taşıyan çekirdek.

Kullanıcı bunu **doğrudan görmez**. Ama şunlar buradan gelir:

- continuity · replay · memory legitimacy  
- governance · drift resistance  

Bu ayrım [invisible depth](RHIZOH_FREEZE_0.md#3-invisible-depth-rhizoh-ux-ana-prensibi) ile aynı hizada.

## 2. Katman II — Rhizoh Product Surface

**Yüzey:** Companion · Chronicle · Observe · Studio — [FREEZE-0 UI collapse](RHIZOH_FREEZE_0.md#2-ui-collapse-castle-genesis), [tek duygusal merkez](RHIZOH_FREEZE_0.md).

Özellikler: **sakin**, anlaşılır, insan merkezli, duygusal ritmi olan, günlük kullanıma uygun. **Epistemic OS** bilinçli şekilde **geri planda**; insan **governance algebra**’ya değil **süreklilik hissine** bağlanır.

**Translation layer:** [RHIZOH_UI_LANGUAGE_GUIDE.md](RHIZOH_UI_LANGUAGE_GUIDE.md) — iç terminolojiyi kullanıcı diline çevirmek; ileri sistemlerin “kendi jargonuna aşık” olma riskine karşı olgun ürün davranışı.

**İlişki mühendisliği:** [RHIZOH_COMPANION_PERSONALITY_STABILITY.md](RHIZOH_COMPANION_PERSONALITY_STABILITY.md) — capability’den çok **personality continuity**.

**Stratejik konumlanma:** [RHIZOH_CALM_TECHNOLOGY_PRINCIPLE.md](RHIZOH_CALM_TECHNOLOGY_PRINCIPLE.md) — dikkat ekonomisine ters, **sessiz güven** tarafı.

## 3. Katman III — Civilization Layer

**İçerik:** Spiral, collective chronicle, simulation, org cognition, robotics köprüsü — **özellik paketi** değil, **uzun vadeli civilization substrate** olarak konumlanır.

[Spiral’i geç açma](RHIZOH_PRODUCT_PHASES_A_THROUGH_E.md) kararı, ürün kimliğinin **“oyun”** olarak okunma riskini düşürür; bu katman medeniyet **yüzeyi** olur, stack’in “gimmick” özeti değil.

## 4. Embodied Product Reality (yeni faz)

Soru artık yalnız *“Spec doğru mu?”* değil — **bedenlenmiş ürün gerçekliği:**

| Soru |
|------|
| Mobilde **nasıl hissettiriyor**? |
| İnsanlar **timeline’a / Chronicle’a dönüyor** mu? |
| Chronicle **anlam taşıyor** mu? |
| **Castle node** kurmak doğal mı? |
| Companion **günlük rutine** giriyor mu? |
| Studio ekiplerde **kalıcı** mı? |
| Sistem **yorucu** mu **sakin** mi? |
| Kullanıcı **güveniyor** mu? |
| İnsanlar bunu **başkalarına gösteriyor** mu? |

Bu **gerçek dünya validasyonu** — [Reference User Journey](RHIZOH_COMPANION_REFERENCE_JOURNEY.md) §0–5 ile birlikte okunur.

## 5. Eşik ve risk

**Olumlu eşik:** Scope sınırlandı, katman sırası net, public language oluştu, ürün kimliği tanımlandı, freeze mantığı ve görünmez derinlik oturdu — repo **tamamlanabilir** hissedilebilir.

**En büyük risk:** Yeniden **sınırsız teori / spec genişlemesine** kaymak.

---

## 6. Rhizoh governance kuralı (spec öncesi zorunlu soru)

**Yeni normatif spec veya çekirdek genişlemesi yazmadan önce** şu soru **zorunlu** olmalıdır:

> **Bu, kullanıcı deneyiminde gerçekten hissedilecek mi?**

- **Evet** → maliyet / sıra / FREEZE ihlali yoksa ilerle.  
- **Hayır** → iç araç, deferred, veya hiç — **teori için teori** genişlemesi yapılmaz.

Bu kural, **Frozen Civilization Kernel**’in yavaşlığını korurken **Product Surface**’in yaşama geçmesini zorunlu kılar.
