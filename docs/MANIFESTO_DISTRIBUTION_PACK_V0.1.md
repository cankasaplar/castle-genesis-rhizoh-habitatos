## §2 — Epistemic Firewall

**Observation affects narrative; never execution.**

- **Observation (O):** read, project, interpret — no write-back to sealed state.
- **Execution (E):** deterministic paths, seals, WAL, frozen subgraph — no upward causality from story.
- **Narrative (D):** rendered under provenance; companions **`executive: false`**.

Lab AI, LLM streams, and external plugins are **untrusted producers** — semantic interpreters, not co-sovereigns. See [`EXTERNAL_LAB_AI_INTEGRATION_SPEC_V1.md`](EXTERNAL_LAB_AI_INTEGRATION_SPEC_V1.md) · EFIR-α layers in [`RHIZOH_CASTLE_GENESIS_PRODUCTION_ARCHITECTURE_V1.md`](RHIZOH_CASTLE_GENESIS_PRODUCTION_ARCHITECTURE_V1.md) §0.

## §3 — No Boot Token From Story

**Story is not ontological proof.**

Narrative output must not mint execution authority, continuity seals, or `bootValidityToken`-class facts. If it is not traceable through **derived → sealed** paths, it does not boot the world.

**Checksum for contributors:** Before wiring a feature, ask: *“Could a convincing paragraph become state?”* If yes, stop — add membrane, provenance, or gate.

**Code discipline:** `narrativeSourceProvenanceV0` · `∀ narrative ⇒ derived` · no C-layer entanglement from D-only input.
